📋 Bản Kế Hoạch Dành Cho Claude Code (Copy phần bên dưới)
Mục tiêu dự án:
Xây dựng một Python Agent để chuyển đổi file tài liệu (PDF, EPUB) thành cấu trúc JSON chuẩn hóa, và tiền xử lý văn bản đó bằng SSML để làm đầu vào (input) cho các hệ thống Text-to-Speech (TTS) tạo Audiobook.

Nguyên tắc cốt lõi:

Ưu tiên xử lý bằng Code logic (Regex, Heuristics, DOM Parsing) cho 80% công việc.

Chỉ dùng LLM (OpenAI/Anthropic API) như một fallback function cho các tác vụ khó (sửa lỗi chính tả do scan, phân loại câu khó).

Code cần theo chuẩn OOP (Object-Oriented Programming) hoặc Functional có typing rõ ràng (Python Type Hints).

Tech Stack yêu cầu:

Ngôn ngữ: Python 3.10+

Xử lý PDF: PyMuPDF (fitz) - Bắt buộc dùng get_text("dict") để lấy metadata font chữ (size, weight).

Xử lý EPUB: EbookLib, BeautifulSoup4.

Tiền xử lý Text: re (Regex), num2words (hỗ trợ đọc số).

🚀 Lộ trình Triển khai (Implementation Plan)
Vui lòng triển khai theo đúng thứ tự 5 Phase dưới đây. Hãy hoàn thành từng Phase, viết Unit Test cơ bản, xác nhận với tôi trước khi chuyển sang Phase tiếp theo.

Phase 1: Setup Project & Interfaces


Định nghĩa Pydantic Models cho dữ liệu đầu ra để đảm bảo tính nhất quán:

DocumentBlock: Gồm level (int, 0 là nội dung, 1-3 là heading), type (enum: chapter, subtitle, body, table_of_content), text (string).

ParsedDocument: Gồm list các DocumentBlock.

Phase 2: Core Extractors (Trích xuất dữ liệu thô)

EPUB Extractor: * Viết class đọc file EPUB.

Lặp qua các item HTML, dùng BeautifulSoup để map các thẻ <h1>, <h2>, <h3> thành level 1, 2, 3 và thẻ <p> thành level 0 (body).

PDF Extractor: * Viết class dùng PyMuPDF đọc PDF.

Trích xuất text theo dạng dictionary (page.get_text("dict")).

Phân tích các span để lấy font_size và flags (bold/italic).

Thuật toán gom nhóm: Tính toán font size trung bình để xác định text bình thường (body). Font size lớn hơn trung bình + bold sẽ được map thành heading level 1, 2.

Phase 3: Text Cleaning & Heuristics (Làm sạch & Nối câu)
Viết một class TextCleaner nhận đầu vào là các blocks từ Phase 2:

Xóa Header/Footer: Dùng Regex và thuật toán so khớp để phát hiện các dòng text giống hệt nhau lặp lại ở top/bottom của nhiều trang, cùng với các con số đơn độc (số trang), sau đó xóa bỏ.

Thuật toán Nối dòng (Line Unbreaking): Nếu block hiện tại là body và kết thúc không phải là dấu chấm/chấm hỏi/chấm than (., ?, !, "), hãy nối nó với block body tiếp theo, thêm khoảng trắng ở giữa.

Lọc bỏ các block quá ngắn mà không chứa ký tự chữ (VD: các dòng chỉ chứa ***, ---, hoặc số).

Phase 4: TTS Pre-processor & SSML Injector
Viết class TTSFormatter nhận input là ParsedDocument:

Chuẩn hóa Text: Viết hàm dùng Regex để chuyển đổi ngày tháng (VD: 12/05/2026 -> ngày mười hai tháng năm năm hai không hai sáu), chuyển số thành chữ (nếu cần), và mở rộng từ viết tắt phổ biến (VD: "đ/c" -> "đồng chí", "TP" -> "Thành phố").

SSML Wrapper: Dựa vào level của block, bọc text bằng thẻ SSML.

Heading 1 (Chapter): <speak><s>...</s><break time="2s"/></speak>

Heading 2 (Subtitle): <speak><s>...</s><break time="1s"/></speak>

Body: Dùng regex tách các câu (split theo dấu chấm) và bọc mỗi câu trong thẻ <s>, thêm <break time="0.5s"/> giữa các đoạn văn.

Phase 5: CLI & Pipeline Orchestration

Tạo file main.py dùng argparse để người dùng có thể gọi lệnh từ terminal.

Lệnh ví dụ: python main.py --input book.pdf --output result.json --lang vi

Tích hợp tùy chọn fallback LLM (tùy chọn): Nếu một đoạn PDF bị mã hóa lỗi (Garbled text), gửi đoạn đó qua API để LLM "dịch" lại thành tiếng Việt chuẩn.
